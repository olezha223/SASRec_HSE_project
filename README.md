# Deep RecSys: Transformer-based Next Item Prediction (Yambda)

Реализация рекомендательной модели отбора кандидатов на основе GPT-style
трансформера: пользовательская история кодируется каузальным энкодером,
модель обучается на задачу предсказания следующего айтема (next item
prediction, NIP), используется global temporal split, 
кандидаты для инференса ранжируются по всему каталогу.

## Датасет

Используется датасет **Yambda** (лайки, версия 500m), а именно файл
`interactions.parquet` - таблица `user-item` взаимодействий с колонками
`uid`, `item_id`, `timestamp`. Дополнительно используются:

- `embeddings.parquet` - только для того, чтобы отфильтровать айтемы, для
  которых есть предрасчитанные эмбеддинги (сами эмбеддинги в модели не
  используются, только множество `item_id`);
- `artists.parquet` - метаданные айтемов (маппинг в артистов), присоединяются
  для полноты данных.

### Фильтрация и подготовка

1. **Semi-join по эмбеддингам** - оставляем только взаимодействия с айтемами,
   которые присутствуют в `embeddings.parquet`.
2. **Item core-фильтрация** - оставляем только айтемы, у которых число
   взаимодействий ≥ `CORE_MIN_INTERACTIONS_PER_ITEM = 5` (отсекаем
   "холодные" редкие айтемы).
3. **Join с артистами** - обогащение метаданными (`artists.parquet`, `left join`).
4. **Ремаппинг `item_id`** - исходные идентификаторы переиндексируются в
   непрерывный диапазон `[0, catalog_size)`.

### Разбиение на train/test (по времени)

- Порог разбиения: `test_start_ts = max(timestamp) - 7 дней`
  (`TEST_INTERVAL_SECONDS = 7 * 24 * 60 * 60`).
- `train_df` - все взаимодействия с `timestamp < test_start_ts`.
- `test_df` - все взаимодействия с `timestamp >= test_start_ts`.
- Таргеты для валидации (`targets`) - это айтемы каждого пользователя из
  `test_df`, при условии, что у пользователя есть непустая история в train.

### Формирование историй и сэмплов

- Для каждого пользователя история сортируется по `timestamp` и превращается
  в список `item_id` (`histories: Dict[uid, List[item_id]]`).
- **`YambdaTrainDataset`**: история пользователя режется на непересекающиеся
  чанки длиной не более `max_seq_len` (сдвиг от начала к концу истории).
  Каждый сэмпл - это пара `history = [i_1 ... i_{L-1}]`,
  `targets = [i_2 ... i_L]` (авторегрессионный сдвиг на 1 позицию),
  плюс `length`. Пользователи с историей короче 2 событий отбрасываются.
- **`YambdaEvalDataset`**: для каждого пользователя, у которого есть таргеты
  в тесте, берутся последние `max_seq_len` событий его истории как контекст
  (`history`), без явных `targets` - они хранятся отдельно и используются
  только на этапе подсчёта метрик.
- **`collate_fn`**: батч собирается в **flatten-представлении** - все истории
  батча конкатенируются в один 1D-тензор, паддинг не используется, границы
  последовательностей восстанавливаются по тензору `length`. Аналогично
  собираются `targets` (только для train).

## Архитектура модели

GPT-style каузальный трансформер (декодер), обрабатывающий историю
взаимодействий пользователя:

1. **`CausalSelfAttention`** - multi-head self-attention с объединённой
   маской: каузальная маска (запрет смотреть в будущее, `torch.triu`) +
   паддинг-маска. Реализовано через `F.scaled_dot_product_attention`.
2. **`MLP`** - position-wise feed-forward: `Linear(d → 4d) → GELU → Dropout →
   Linear(4d → d)`.
3. **`Block`** - трансформер-блок в Pre-LayerNorm постановке: два
   residual-подблока `LN → Attention → Dropout → +residual` и
   `LN → MLP → Dropout → +residual`.
4. **`GPT`** - стек из `n_layers` блоков `Block` + финальный `LayerNorm`.
5. **`UserEncoder`** - принимает flatten-историю и `length`:
   - lookup эмбеддингов айтемов (`nn.Embedding`, общий для истории и
     таргетов/негативов - единая айтемная башня);
   - восстановление padded-тензора и маски через `create_masked_tensor`;
   - добавление позиционных эмбеддингов;
   - прогон через `GPT`;
   - обратное превращение в flatten-представление (`encoded[mask]`).

Модель - **двухбашенная**: пользовательская башня (`UserEncoder`) строит
контекстное представление префикса истории, айтемная башня - это одна общая
матрица `nn.Embedding`, используемая и для входных айтемов, и как таргетные/
негативные представления при подсчёте лосса.

### Обучение - `TrainNIPModel`

- Задача: **next item prediction** - по скрытому состоянию на позиции `t`
  предсказывается айтем на позиции `t+1` (сэмплы уже сдвинуты на этапе
  датасета).
- **Негативы**: in-batch негативы - для каждого позитива случайно
  выбираются `num_negatives` таргетов из этого же батча (без явного
  исключения совпадений с позитивом).
- **logQ-коррекция**: логиты негативов корректируются на лог-частоту айтема
  в train-таргетах (`logq(i) = log(count(i)) − log(Σ count)`), чтобы
  компенсировать смещение in-batch-сэмплирования в сторону популярных
  айтемов. Частоты (`q_counts`) считаются заранее по всем `targets`
  train-датасета (`build_q_from_train_targets`) и хранятся как буфер модели.
- **Лосс**: логиты позитива и скорректированных негативов конкатенируются
  и подаются в `cross_entropy` с меткой класса 0 (позитив всегда первый) -
  эквивалент sampled softmax / in-batch negative sampling с logQ-коррекцией.
- Инициализация весов - усечённое нормальное распределение (`std=0.02`).

### Валидация - `EvalNIPModel` + `eval`

- `EvalNIPModel` прогоняет `UserEncoder`, для каждого пользователя берёт
  скрытое состояние **последнего** события истории и считает скалярное
  произведение с эмбеддингами всех айтемов каталога → матрица скоров
  `[B, num_items]`.
- Функция `eval`:
  - переводит модель в `eval()`, использует `torch.no_grad()`;
  - для каждого батча берёт `top-k` айтемов по скору (`torch.topk`);
  - собирает предсказания в `Dict[uid, List[item_id]]`;
  - по всем пользователям вызывает `evaluate(...)` и возвращает финальные
    метрики.

## Метрики

Считаются по каждому пользователю и усредняются по всем пользователям
(`topk = 100`):

- **HitRate@k** - попал ли хотя бы один релевантный айтем в top-k.
- **Recall@k** - доля найденных релевантных айтемов относительно
  `min(k, |targets|)`.
- **NDCG@k** - normalized discounted cumulative gain с идеальным DCG,
  рассчитанным по числу релевантных таргетов.
- **Coverage@k** - доля уникальных айтемов каталога, попавших хоть в чьи-то
  рекомендации, от общего размера каталога.

## Основные гиперпараметры

| Параметр | Значение |
|---|---|
| `embedding_dim` | 64 |
| `n_layers` | 2 |
| `n_heads` | 2 |
| `dropout` | 0.1 |
| `max_seq_len` | 100 |
| `num_negatives` (in-batch) | 512 |
| `TRAIN_BATCH_SIZE` / `EVAL_BATCH_SIZE` | 128 |
| `NUM_EPOCHS` | 10 |

| Test-интервал | последние 7 дней взаимодействий |
